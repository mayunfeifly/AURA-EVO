# AURA EVO — Chrome Extension Limitations

> 插件不能实现的功能清单，后续网站版补充

## 必须网站版实现的功能

| # | 功能 | 为什么插件不行 | 优先级 |
|---|---|---|---|
| 1 | **模型训练/蒸馏** | 需要 GPU 算力，浏览器无法运行 LoRA/QLoRA | P0 |
| 2 | **文件上传（大文件）** | 扩展 storage 限制 5-10MB | P0 |
| 3 | **专家包交易/支付** | Chrome Web Store 支付抽成 30%，且流程复杂 | P0 |
| 4 | **创作者后台** | 需要复杂数据看板，扩展界面太小 | P1 |
| 5 | **多设备同步** | 扩展数据绑定单设备，需服务器中转 | P1 |
| 6 | **用户间消息/社交** | 扩展无实时通信能力 | P2 |
| 7 | **复杂数据分析** | 扩展计算能力有限 | P2 |

## 插件可以实现但体验受限的

| # | 功能 | 插件限制 | 临时方案 |
|---|---|---|---|
| 1 | **知识库对话** | 上下文长度受限 | 分段检索，精简提示 |
| 2 | **页面内容提取** | 跨域限制 | 当前页 only |
| 3 | **本地文件读取** | 需用户手动选择 | 文件选择器 |
| 4 | **后台常驻** | Service Worker 会休眠 | 定时唤醒 |

## MVP 阶段策略

```
插件 = 入口 + 轻交互 + 页面感知
网站 = 重功能 + 交易 + 训练 + 创作者后台
```

**Phase 1（现在）：**
- 插件：知识库上传（小文件）、预设专家对话、页面内容提取
- 网站：Landing Page 展示

**Phase 2（有收入后）：**
- 网站：真蒸馏、专家市场、支付、创作者后台
- 插件：调用网站 API，做轻量入口

**Phase 3：**
- 多平台统一账号体系
